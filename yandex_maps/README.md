# Анализ динамики метрик Яндекс Карт

## Цель проекта:
Построить масштабируемую аналитическую систему для анализа исторических метрик Яндекс Карт:
- выявление трендов и сезонности
- расчет годовой динамики (YoY)
- подготовка отчетных материалов для команд продаж рекламы

## Исходные данные
Исторические метрики активности пользователей Яндекс Карт
Разрезы:
- индустрия → подгруппа → рубрика
- тип метрики
- помесячные значения за несколько лет

## Этап 1. Прототипирование и согласование логики (Excel + Power Query)

Цель этапа: быстро согласовать бизнес-логику расчетов с заказчиком.

Реализовано:

Очистка и нормализация данных
Извлечение года и месяца
Переход от сырых значений к аналитическим показателям
Реализованные расчеты:
- Группировка данных по календарным годам
- Помесячная агрегация метрик:
   суммирование для кумулятивных показателей
- Расчет YoY:
   (текущий год – предыдущий год) / предыдущий год * 100%
- Расчет сводных статистик:
  * среднее значение
  * минимум и максимум
  * средний YoY

На этом этапе был собран первый дашборд в DataLens для обсуждения с заказчиком.

## Этап 2. Перенос в БД и масштабирование (PostgreSQL + Supabase)

После утверждения логики расчеты были перенесены в базу данных:

Архитектура решения:
raw_maps — таблица хранения сырых данных
SQL-представления:
   - приведение данных к «длинному» формату
   - расчет агрегатов и статистик
   - формирование финальной витрины для BI
Ключевые решения:
Расчеты выполняются на стороне БД → быстрый отклик дашбордов
Новые месяцы добавляются без пересчета истории
Подключение DataLens напрямую к финальной витрине

Дашборд (Yandex DataLens)
Реализовано:
- Фильтры по индустриям, рубрикам и метрикам
- Сравнение значений по годам
- Вывод YoY и агрегированных показателей
- Использование дашборда как инструмента продаж

## Бизнес-результат:
- cокращение времени подготовки отчетности и презентаций с нескольких дней до минут
- увеличение числа аналитических срезов с 10 до 43
- автоматизация обновления данных и повышение качества данных для принятия решений
